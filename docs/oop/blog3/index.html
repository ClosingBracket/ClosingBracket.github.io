CS371p Fall 2020: Tony Chang - Week 3

CS371p Fall 2020: Tony Chang #

What did you do this past week?

This week, I worked to first implement the simplest possible working solution of Collatz. After I was able to confirm that was working, I then implemented the multiplicity of optimizations until my code was able to pass all three HackerRank tests. Afterwards, I followed subsequent steps on the workflow such as making my own unit tests to confirm code functionality and my own acceptance tests for merging on the public test repo. From there, I generated the documentation using doxygen and passed my code through astyle.

What’s in your way?

I’ve currently finished the project, so nothing is currently in my way at the moment! I was able to learn a lot about GitLab issues, the CI pipeline, and C++ code development.

What will you do next week?

For OOP, I don’t have much left to do thankfully for Collatz. I’ll continue attending lecture, taking the quizzes, and working on the weekly assignments such as Perusall and these blog posts.

What was your experience of Collatz, the starter code, the makefile, its optimizations, and exceptions?

I enjoyed the project. It was a nice dip-your-feet-in experience for C++ because the majority of the IO was handled through the starter code such as collatz_read and such. As a result, we only needed to handle the actual computation itself. Working through the starter code was fine, because it was well-formatted and not too many lines of code. I thought the makefile on the other hand was quite big and complex, because very seldom actually take a peek at the makefiles I use and usually just type make clean and make run test or such. As a result, I’m not too quite familiar with the syntax of it all.

I thought the optimizations that Prof. Downing mentioned to optimize our Collatz running time are very clever. Some are more straightforward such as the if odd, simply do even calculation right there too but others such as the max_cycle_length(b, e) = max_cycle_length(i, e) if condition was very useful.

I think exceptions are incredibly handy. I always knew they existed through classes such as CS314 where you must throw an exception if a precondition wasn’t met for example, but I’ve never formally learned exactly how to catch them or what they are, so I thought that class was beneficial.

What made you happy this week?

This is related to my item from last week, but I very much enjoyed the four day week. Mondays for me are quite busy with many classes, so having that slight break was nice.

What is your pick-of-the-week?

My pick of the week is vim! vim is an text editor designed for use from a command-line interface, so it is incredibly useful for example, when you need to SSH into a machine and make some quick edits to a file. You don’t need to go through your normal workflow of potentially opening up your IDE, making the edit, and pushing the changes onto some server. With vim, you can simply do vim filename and make the edit on the server! Just remember that you need to type :wq to save and exit.
