CS371p Fall 2020: Tony Chang #

What did you do this past week?

This week, I learned about throwing and catching exceptions in C++, gained more understanding on the Voting project that we have recently been assigned, and string comparison in C++.

What’s in your way?

I feel intimidated by the scale of the Voting project. In the last project, we were given the run and test harness and only needed to handle the actual Collatz logic itself. However, in this project, we need to build up the entire project from the start. I’ve never worked with C++ cin and cout so this will be a unique challenge for me to figure out.

What will you do next week?

This next week will be very busy for me, so I am afraid that I won’t have as much time as I want to work on the project. I definitely need to start by making the GitLab repo, defining my issues, and most likely importing some of the items from last week’s project such as the Makefile and the ci.yaml file. From there, I will need to make sure that the data is being read correctly from the file into the program.

What was your experience of exceptions, voting, and strcmp?

I think exceptions are a valuable tool for error handling. Assertions are very serious because they will halt the entire program if they appear, which will not always be the desired behavior. If we want to handle user error for example, it would be better to throw an exception and catch it somewhere higher up in order to gracefully handle the error. The voting system aforementioned in class too seems intuitive for a multi-candidate or multi-party election systems (which the United States is definitely not). It ensures that when only a plurality is achieved, we will be able to choose the candidate that most people will prefer as best as we can. strcmp is definitely a very handy function and actually something we implemented in CS429 String lab, so I was already familiar with what strcmp does and how it works from implementation. The exercise felt familiar to me so I don’t have much to write about that.

What made you happy this week?

I’ve had opportunities to see my friends this week through online mediums such as Zoom and Discord! It’s not the same as being in-person but I’m glad that we’re still able to find ways to catch each other online regardless.

What is your tip-of-the-week?

My tip-of-the-week is a multi-monitor setup! I have personally found that in this semester of online classes where we have to consult a lot more video and text resources online, I have found that a multi-monitor has been dramatically beneficial to my productivity. Being able to work on code on one screen and look at project guidelines or resources on the other means that you don’t need to spend time switching windows and going back-and-forth.
